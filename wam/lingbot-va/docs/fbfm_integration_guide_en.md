# FBFM (Flow-Based Feedback Model) Integration Guide

This document summarizes the core code modification strategies, architectural design, and critical development details for integrating the FBFM strategy into the WanVA server side.

## 1. Core Modification Strategies

### 1.1 Scheduler Enhancement: `WrapperedFlowMatchScheduler`
To support state correction based on feedback, the original `FlowMatchScheduler` is wrapped into `WrapperedFlowMatchScheduler`. Its core logic involves inserting a **gradient correction step** within the standard denoising process.

- **Virtual Time Axis Mapping**: Maps the standard Diffusers Sigma time axis to the $\tau$ ($ \tau = 1 - \sigma $) used by RTC (Residual Time Coordinate) to align with FBFM theoretical formulas.
- **Dynamic Correction Term Calculation**:
  - When constrained states (`constrained_y`) and weights (`weights`) exist, `torch.enable_grad()` is enabled.
  - Calculates the predicted $x_1$ ($x_t - \tau \cdot v_t$).
  - Computes the error $err = (y_{target} - x_1) \cdot weights$.
  - Backpropagates the error to $x_t$ via `torch.autograd.grad` to obtain the correction direction `correction`.
- **Adaptive Guidance Weight**:
  - Introduces time-dependent gain coefficients $c(\tau)$ and $inv\_r2$ to suppress excessive corrections early in the diffusion process ($\tau \to 1$), preventing divergence.
  - Applies truncation using `max_guidance_weight` to ensure numerical stability.
- **Final Velocity Field Correction**: $v_{result} = v_t - \text{guidance\_weight} \cdot \text{correction}$.

### 1.2 Generic Denoising Function Abstraction: `get_denoise_fn`
To reduce code duplication and unify logic for Video and Action generation, the `get_denoise_fn` higher-order function is extracted.

- **Mode Unification**: Automatically selects input dictionary keys (`latent_res_lst` / `action_res_lst`) and post-processing logic via the `mode` parameter ('video' / 'action').
- **CFG Logic Cohesion**: Encapsulates the Classifier-Free Guidance calculation logic ($noise_{cond} + scale \cdot (noise_{cond} - noise_{uncond})$) within the closure, avoiding repetitive conditional checks in the inference loop.
- **Differentiable Interface**: The returned `denoise_fn` maintains the complete computation graph to allow gradient backpropagation from the outer Scheduler.

### 1.3 Feedback Mechanism Integration (`_feedback` & `PrevChunk`)
A state feedback queue mechanism is introduced in `VA_Server` to handle continuity constraints during multi-chunk generation.

- **State Caching**: The `_feedback` method receives real observed states from the previous frame, encodes them into Latents, and stores them in `self.prev_chunk_left_over` (the FBFM strategy object).
- **Constraint Injection**: During the `_infer` loop when calling `scheduler.step`, `constrained_y` (target state) and `weights` (spatial/channel weights) are extracted from the strategy object and passed to the scheduler for real-time correction.

## 2. Critical Details to Note

### 2.1 Gradient Graph Management
- **Explicit Gradient Enablement**: In `WrapperedFlowMatchScheduler.step`, the correction calculation block must be wrapped with `with torch.enable_grad():`, as inference mode defaults to `no_grad`.
- **retain_graph=False**: Set `retain_graph=False` in `torch.autograd.grad` because corrections at each timestep are independent; retaining the entire diffusion graph is unnecessary and saves VRAM.
- **Detach Operations**: Input `x_t` must be `.clone().detach()` before correction to ensure gradients flow only within the correction block and do not contaminate upstream generation history.

### 2.2 Time Axis and Sigma Boundary Handling
- **Tau Calculation**: Pay attention to the definition $\tau = 1 - \sigma$ to ensure consistency with FBFM paper or implementation symbols.
- **Final State Handling**: When `to_final=True` or at the last step, $\sigma_{next}$ should be forced to 0 (or 1, depending on the inverse process definition). FBFM correction is typically skipped at this stage, outputting the result directly.
- **Padding Alignment**: The code performs `F.pad(..., value=0)` on `timesteps`. Ensure the sequence length generated by `set_timesteps` strictly matches the inference loop to prevent index out-of-bounds errors.

### 2.3 CFG and Batch Dimensions
- **Batch Repetition**: When `guidance_scale > 1`, `_repeat_input_for_cfg` doubles the input batch dimension (Condition + Uncondition).
- **Slicing Logic**: In `get_denoise_fn`, after CFG calculation, correctly slice (`noise_pred[:1]` or `noise_pred[1:]`) to ensure the output dimension returns to the original Batch Size; otherwise, subsequent matrix operations will fail.
- **Cache Consistency**: When creating the Transformer KV Cache, dynamically adjust the `batch_size` parameter (1 or 2) based on `use_cfg`; otherwise, cache hits will fail or shape mismatches will occur.

### 2.4 Action Space Masking
- **Zeroing Invalid Channels**: After the generation loop, execute `actions[:, ~self.action_mask] *= 0` to ensure untrained action channels (e.g., fixed base positions) remain zero, preventing the model from generating hallucinated noise.
- **Normalization Consistency**: Quantile normalization (`q01`, `q99`) in `preprocess_action` and `postprocess_action` must strictly match the statistics used during training; otherwise, the correction direction will deviate from the true physical space.

### 2.5 VRAM Optimization
- **Offload Strategy**: Depending on `enable_offload`, VAE and Text Encoder can be placed on CPU during initialization and temporarily moved to GPU only during inference.
- **Cache Cleanup**: At the end of each Chunk inference or upon Reset,务必 call `transformer.clear_cache` and `torch.cuda.empty_cache()` to prevent OOM during long-sequence inference.

## 4. Configuration Recommendations

It is recommended to add the following parameters in `configs` to control FBFM behavior:
- `rtc_config.max_guidance_weight`: Upper limit for maximum correction weight (default suggested 5.0~10.0).
- `action_guidance_scale`: Independent CFG coefficient for action generation.
- `constrain_mode`: Constraint mode (e.g., "Feedback", "Reference").
